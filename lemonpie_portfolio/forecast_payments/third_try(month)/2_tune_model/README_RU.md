# Настройка catboost при прогнозировании месячных поступлений 

В данном разделе настраиваем параметры отбора фондов для прогнозирования в связи с тем, что осталась всего одна модель catboost вместо ансамбля и она не так требовательна к количеству периодов (в отличие от sarimax), а также подбираем количество наборов гиперпараметров для наиболее эффективного прогнозирования поступлений по отдельным фондам (для каждого фонда обучается своя модель прогнозирования).

Набор данных для этапов 1 - 3 все платежи до 01.01.2026 (предобработка одинаковая у всех скриптов).

На **первом этапе** - для проверки результативности подбора параметров отбора фондов добавили расчет метрик по медианным значениям - как наивный прогноз для фондов, по которым не хватает данных для прогнозирования с помощью модели - если прогноз по фондам с помощью модели дает меньшие ошибки, чем простая медиана за предыдцщий период, то значит модель для этого фонда эффективнее.   
Изначально параметры отбора были:  
 - наличие платежей за 3 последних месяца, сделали 4 месяца;
 - наличие всего 18 месяцев с платежами, сделали 6 месяцев;
 - также был фильтр по стартовым значениям ряда - чтобы за первые 6 мес было как минимум 3 не нулевые и таких 6-месячных окон должно было быть как минимум 2 - и окно двигалось с первого значения, пока не выполнялось условие, весь "начальный" хвост отбрасывался, сейчас сделали соответствующие пороги 3-1-1.

**Выводы по этапу**:
Метрики по «старым» 23 фондам остались абсолютно идентичными (различия в пределах погрешности округления). Это подтверждает, что добавление новых фондов не «ломает» обучение старых, так как модели строятся независимо.  
Эффективность на новых фондах: Главный критерий успеха — catboost_gain (разница между ошибкой Медианы и CatBoost). Если значение положительное, модель работает лучше наивного прогноза.  
Успешные кейсы (Модель > Медиана):  
Было - (из 23 фондов):  
	- CatBoost победил: 16 фондов (~70%)  
	- Медиана победила: 7 фондов (~30%)  
Стало(из 63 фондов):  
	- CatBoost победил: 47 фондов (~75%)  
	- Медиана победила: 16 фондов (~25%). 

При расширении воронки (с 23 до 63 фондов) доля побед CatBoost даже выросла (с 70% до 75%). Это говорит о том, что среди "отсеянных" ранее фондов было много таких, чье поведение CatBoost предсказывает гораздо лучше, чем простая медиана. Вероятно, это растущие или меняющиеся фонды, где медиана (ориентируясь на прошлое) систематически ошибается, а бустинг улавливает динамику.
В виду вышеиуказанного - ослабление фильтров полностью оправдано. Хотя на некоторых «рваных» рядах (как 1110) модель проигрывает медиане, суммарный выигрыш на крупных фондах (722, 1054) многократно перекрывает локальные неудачи. Мы начали прогнозировать деньги там, где раньше просто игнорировали фонд.

На **втором этапе** для расширенного набора фондов - добавили набор гиперпараметров, чтобы лучше подстраиваться под большее количество фондов (контролируя переобучение). А именно, добавили глубину 1 (очень простые деревья), увеличили L2 regularization до 10, добавили вариации learning_rate. 
**Вывод по этапу**: Расширенная сетка работает эффективнее базовой в 70-80% случаев. Особенно сильно она помогает фондам со средней историей (как id 176), находя для них более простые и устойчивые модели (например, глубины 1).
Единственный серьезный "выброс" — фонд 722, но в среднем по больнице (по сумме RMSE всех фондов) расширенная сетка выигрывает.

На **третьем этапе** выяснили, что скрипт криво агрегировал месячные данные (де-факто по первому дню месяца, на суммой всех месячных платетежей), поправили ошибку, переобучили модель на тех же данных до 01.01.2026, пересчитали метрики - comparison..ver_6 - c теми же гиперпараметрами как на этапе 2, comparison..ver_7 с гиперпараметрами подобранными уже на месячных данных. По итогу comparison..ver_6 - на корректных месячных данных метрики улучшились, в виду большей стабильности месячных сумм, при расширенном наборе гиперпараметров в comparison..ver_7 зaфиксирован прогресс на большинстве фондов.


Файлы в папке:
 - **comparison...act_restrictions.csv** - метрики исходного набора параметров, с тестом на двух последних месяцах (отбиралось 23 фонда для прогнозирования с помощью модели);
 - **comparison...new_restrictions.csv** - метрики ослабленного набора параметров, с тестом на двух последних месяцах (отобралось 63 фонда для прогнозирования с помощью модели);
 - **comparison...hyperparameters_1.csv** - метрики расширенного набора гиперпараметров для набора фондов с учетом ослабленноых порогов отбора (см.выше);
 - **comparison...ver_6.csv** - метрики на корректно агрегированных месячных данных;
 - **comparison...ver_7.csv** - метрики на корректно агрегированных месячных данных и расширенном наборе гиперпараметров.
 - **comparison...ver_10.csv** -  от версии 7 отличается только добавленными данными - январь 2026, то есть плюс 1 месяц обучения и сдвиг теста на декабрь-январь 2026
