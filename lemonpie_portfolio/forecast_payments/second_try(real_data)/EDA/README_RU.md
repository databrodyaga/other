# EDA платежей

Анализируем данные по платежам от доноров в адрес фондов:

- проводим предподготовку и EDA выгрузки из БД с реальными данными,
- агрегируем данные отдельных платежей до уровня дней с учетом последующего прогнозирования следующего дня, 
- генерируем и добавляем признаки, 
- проверяем корреляции. 

Файлы в папке:
 - **second_try_eda.ipynb** - ноутбук с анализом  
 - **request_1.sql** - запрос к базе (набор сырых данных **data_1.csv** не загружен из-за размера)
 - **users_actual.csv** - действующие user_id (фонды) для фильтра сырых данных
 - **data_with_purpose_mean.parquet**, **lang_model_test.parquet**, **phik_correlation_matrix_full.parquet** - кэш-файлы с расчетами эмбеддингов языковых моделей для кодирования текстовых полей и таблицы корреляции.

Этапа дополнительных исследований и тестов:
- **1_2model_vs_3model_choice** - сравнение двух пайплайнов обучения(с временной составляющей и без) 
- **2_funds_user_id_choice** - анализ порога для отбора фондов для прогнозирования 
- **3_feature_selection** - отбор признаков для итоговой модели
