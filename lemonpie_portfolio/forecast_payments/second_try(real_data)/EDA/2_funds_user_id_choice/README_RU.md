# Отбор фондов

Анализируем метрики по всем фондам (на базе тестовой выборки), чтобы подобрать порог для отбора фондов по которым имеет смысл обучать модели и делать прогноз: если данным слишком мало - модели или дают ошибки (пустые батчи с данными при разбиении обучающей выборкеи) или показывают результаты 
худше наивного прогноза.

Для этого собрали PY-скрипт, чтобы прогнать обучение одинакого пайплайна по всем фондам, с отбором по трем параметрам:  
- абсолютное количество платежей за все время,
- количество дней с платежами (активные дни),  
- соотношение платежей к дням(тут все дни, включая нулевые дни с пропусками активности - чтобы посмотреть среднее количество платежей за весь период работы),
- комбинированные из вышеуказанных параметров.

**Вывод**:  
Лучшее значение по основной метрике RMSE (чем меньше среднее значение по всем фондам - тем лучше) показал порог - минимум 150 активных дней(без учета нулевых неактивных дней), ее оставляем.


Файлы в папке:
 - **second_try_3_models.py** - скрипт с обучением и прогнозами  
 - **request_3.sql** - запрос к базе для выгрузки данных
 - **users_actual.csv** - действующие user_id (фонды) для фильтра сырых данных
 - **2try_data_with_purpose_mean.parquet** кэш-файлы с расчетами эмбеддингов языковых моделей для кодирования текстовых полей и таблицы корреляции.
 - **comparison...csv** - файлы с метриками для разных порого
 - **comparison.xlsx** - сводная таблица с мпетриками на базе исходных csv (см. выше)
