# ML-раздел проекта LemonPie

Раздел содержит модели машинного обучения и скрипты для различных задач системы LemonPie.

## Структура проекта

### `forecast_payments/`
Модели прогнозирования входящих платежей для НКО.

- **first_try/** - первые эксперименты с различными моделями (SARIMAX, LSTM, CatBoost)
- **second_try(real_data)/** - работа с реальными данными
  - **EDA/** - исследовательский анализ данных, выбор признаков
  - **training_models/** - обучение моделей прогнозирования

### `universal_categories/`
Классификация платежей по универсальным категориям согласно классификации Минюста.

- **uc_data_labeling/** - разметка данных для обучения
- **gpt_classification/** - тестирование классификации с помощью публичных LLM (ChatGPT, Gemini, YandexGPT)
  - **ver_1** - версия 1
  - **ver_2-ver_8** - последующие версии с улучшениями
- **own_classification/** - собственные модели классификации
  - **train_test/** - эксперименты с обучением моделей
  - **prod/** - production версия модели

### `yandex_ai_studio_rag/`
Настройка LLM с RAG (Retrieval-Augmented Generation) для чата технической поддержки.

- **rag_ver_1/** - первая версия на основе библиотеки YandexCloudML
- **rag_ver_2/** - вторая версия на основе Responses API (OpenAI-совместимый интерфейс)

## Технологии

- **Python** - основной язык разработки
- **Pandas, NumPy** - работа с данными
- **Scikit-learn** - классические ML модели
- **CatBoost, XGBoost** - градиентный бустинг
- **Transformers (Hugging Face)** - BERT модели для текста
- **YandexGPT, ChatGPT, Gemini** - LLM для классификации
- **Jupyter Notebooks** - исследовательский анализ и эксперименты

## Основные задачи

1. **Прогнозирование платежей** - предсказание входящих платежей на основе исторических данных
2. **Классификация по универсальным категориям** - автоматическая категоризация платежей
3. **RAG для техподдержки** - интеллектуальный помощник на основе базы знаний

## Важные замечания

- Все файлы с данными для обучения содержат только примеры (одна строка)
- Для работы с моделями необходимо настроить окружение и установить зависимости
- API-токены в коде заменены на плейсхолдеры (`YOUR_API_TOKEN`, `YOUR_YANDEX_CLOUD_TOKEN`, `YOUR_FOLDER_ID`). На проде учётные данные передаются через переменные окружения (`os.environ`)

## Установка зависимостей

```bash
pip install pandas numpy scikit-learn catboost transformers torch
```

Для работы с Yandex Cloud ML:
```bash
pip install yandex-cloud-ml-sdk
```

Для работы с OpenAI-совместимым API:
```bash
pip install openai
```
