# Настройка LLM с RAG через ResponsesAPI(OpenAI)

Текущая версия подключения в связи с грядущим переходом Yandex Cloud AI на общепринятый формат работы с AI-агентами и подлючения инструментов для RAG. В данной версии следующие изменения:    
- изменена схема предобработки текстов для базы знаний - помимио размещения сырых текстов расшифровок обучающих видеоматериалов, протестировано и принято решение предобрабатывать тексты с помощью LLM в формат *.jsonl рекомендованной схемы { "body": "текст" } для более эффективного поиска по базе знаний;  
- помимо возможности работы с файлами базы знаний и построения поисковго индекса через web-интерфейс подготовлены управляющие скрипты для осуществления всех действий напрямую по API, тем более что эффективная работа с jsonl файлами реализована только так;  
- перестроен запрос к LLM c учетом изменения архитектуры api, c добавлением диагностики подключенных инструментов RAG (file_search, web_search);  
- схема взаимодействия чата с LLM не изменилась, база знаний регулярно пополняется экспертами компании.  

Состав папки:
- **.md, .jsonl** - файлы базы знаний;
- **convert_jsonl_to_body.py** - скрипт трансформации свободного jsonl в рекомендованную схему { "body": "текст" };
- **index_manager.py** - скрипт управления файлами базы знаний и поисковым индексом (загрузка/удаление файлов разных типов, генерация/удаление индекса);
- **upload_jsonl_to_index.py** - упрощенный скрипт загрузки файлов jsonl рекомендованной схемы в поисковый индекс (как часть функции общего менеджера, см. выше);
- **zapros_rag_2.py** - скрипт запроса к LLM.
