# Классификация универсальных категорий по прогнозам публичных LLM

Собраны оценки прогнозирования разных LLM c учетом дообучения и без. 
В каждой версии набор ключевых файлов следующий:

*Промт для классификации.docx* - не везде есть, т.к. в поздних версиях не использовали стандартные интерейфесы gpt  
*gpt_dataset.ipynb* - разбиение и предобработка датасета  
**gpt_metrics_5000.ipynb** - **расчет метрик** по размеру тестовой выборки (основной файл с комментариями и метриками)   
*yandexgpt_classify.ipynb* - получение прогнозов от стандартной YandexGPT  
*yandexgpt_ft_classify.ipynb* - получение прогнозов от дообученной YandexGPT  
*X_test_balanced.csv, y_test_balanced.csv,*  
*X_train.csv, y_train.csv,*  
*X_test.csv,y_test.csv* - датасеты (версии с "_1","_id" использовались для синхронизации и сверки с более поздними наборами платежей по id)  
*y_pred_chatgpt.csv* - прогнозы ChatGPT  
*y_pred_gemini.csv* - прогнозы Google Gemini  
*y_pred_ygpt.csv* - прогнозы YandexGPT  
*y_pred_ygpt_ft.csv* - прогнозы дообученной YandexGPT  
**gpt_classification_prod.py** - продакшен версия скрипта загрузки данных, получения меток и загрузки в БД сервиса.

Разные этапы тестирования:  
**ver_1** - ошибочно дообучили модели на 5000 примеров одного класса, эффективность низкая;  
**ver_2** - корректно дообученные модели по данным ТОЛЬКО текстов назначений платежей, метрики chatgpt - 75%, yagpt - 70%;  
**ver_3** - попытка дообучить на еще более смещенном по классам датасете, не удалась, метрики низкие;  
**ver_4** - тест yagpt, дообученной для версии 2, на другом тестовом наборе - метрики пдтвердились;  
**ver_5** - дообучали LLM на данных с добавление названий статей, проектов и доноров, тестировали на том же наборе что и ранее,   
        LLM тупо выучила распределние по допназваниям, в итоге метрики 100%;  
**ver_6** - 50% тестового датасета (по половине примеров для каждого класса) было скорректировано (добавлены опечатки, аналогии, пропуски),  
        искажения значительные и yagpt все равно показал метрики 96%;  
**ver_7** - протестировали на модели для версии 5 исходный тестовый датасет с текстами только назначений платежа, метрики упали до 30% - без дополнительных данных модель не   справляется;  
**ver_8** - протестировали на модели для версии 5 датасет реальных платежей, с новыми значениями статей, которых ранее модели не видели, на 1721 пример с дисбалансом классов, из аналогичных тестов 4 и 6 собственной модели.  Результат интересный, но не выдающийся.
